では、まずはclaude codeの説明から行います。
Claude Codeは、“コードを書いてくれるチャット相手”みたいなAIです。
導入には、node.js と、npm コマンドが使える環境が必要です！
また、claude code 自体、導入にお金がかかるのですが、月数百円程度でけっこう使えるので、試しにやってみたい！という方にもおすすめです！
claude code が普通のチャットボットと違うところは、
      * プロジェクト全体のファイルを見てくれる

      * 新しいファイルをまとめて作ってくれる

      * エラー文を投げると、原因と直し方を説明してくれる
といったコード全体のアシスタントをしてくれるところです。


ですので、
   * コードを100%理解しなくても大丈夫。7割分かれば上等、残りはAIに聞く

   * 知らない単語は、そのままClaude Codeに『これ何？』って聞けばOK！

という使い方で大丈夫です！




Remotionは、“ReactっていうWeb技術で動画を作るツール”です。
でもreactが何かについてはとくにわかってなくても大丈夫！
ふつうの編集ではマウスでタイムラインをいじりますが、remotionでは、「何秒〜何秒に何を出すか」をコードで書く、というふうに考えてくださいね。
動画編集初心者としてありがたいのは、
      * いったんテンプレができたら、テキスト差し替えで量産できる

      * 解説動画・ニュース・商品紹介など“同じ型で大量生産”に強い

ってところ。


ここまで使うツールについて説明してきたけど、

Remotion：動画を動かす「土台」

Claude Code：その上に乗せる「コード」を書いてくれる担当

と覚えておいてくれるとうれしいな！


ここではコードによる動画編集に必要なものを列挙していくよ。
まず、PC（Windows / Mac）。それに、
         * Node.js

         * コードエディタ（VS Codeなど）

         * Claude Code を使えるブラウザ環境

         * Remotion のスタータープロジェクト

が必要になってきます。
ここを細かくやるとそれだけで30分動画になるので、今回は“地図”としてのざっくり説明だけにとどめます。質問があったらコメントしてね！
ざっくりの流れとしては、
            * Node.jsを公式サイトからインストール

            * ターミナル（コマンドプロンプト）でRemotionプロジェクト作成

            * npm start でプレビュー
といった感じになります。
コマンドの意味は分からなくてOK。“コピペして動けば勝ち”くらいで大丈夫！！


これがこれから行うフローの全体像だよ。
このあとのパートで、②〜④をミニデモで見せます！
全部理解する必要はなくて、“流れがイメージできたら勝ち”と思って見てね。


6-1.まず日本語で仕様を決める（5:40〜6:10）
ここで意識するのは、いきなりコードじゃなくて、“日本語で仕様を書く”ところから始めます。
例として決める内容を骨子だけ書き出すとこんな感じ。

               * 尺：30秒前後

               * 構成：

                  * 0〜5秒：タイトル

                  * 5〜26秒：ポイント3つを順番に表示（各7秒）

                  * 26〜30秒：まとめ
                  * 背景色・文字色：落ち着いた見やすい配色


さっきの仕様を、そのままAIへの“お願い文章”に変えていきます。
含めるポイントは以下の通りです。

                     * Remotionで30秒の動画コンポーネントを作ってほしい

                     * 秒数ごとに表示するテキストを指定

                     * 背景・フォントサイズなども希望を書く

                     * 「React初心者なのでコメント多めで」と頼む

プロンプトは箇条書きで書くと真似しやすいですよ！
さて、ここまで書くとclaude code がコードを生成してくれます。けっこう長いコードができますが、生成されるコードをすべて理解する必要はありません。
remotionのスタジオ機能でプレビューが確認できるので、そちらをみながら調整していきましょう。
文字の色やサイズを変えたい場合は、右の窓にpropsという設定画面があるので、そこを直接いじって変更ができます！
ちなみにこの窓はカスタマイズ可能なので、理解できてみたらプロンプトでお願いしてみて下さいね！


はじめて環境を作ると、正直なんかしらエラー出ることが多いです。
そういうときも焦らずにいっこいっこ対応していきましょう！
エラーが出たら、エラー文面をそのままターミナルにコピーして、クロードに聞いてみましょう。
一番大変なのは“最初の1本目”だけで、
2本目からはファイルをコピーして、プロンプトを差し替えすることで量産できます！
